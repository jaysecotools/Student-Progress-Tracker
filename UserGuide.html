Introduction

The Vocational Education Progress Tracker is a web-based application designed for educators and administrators to monitor and manage student progress through certificate-based courses. It provides a centralized platform to track individual student task completion, manage course structures, visualize overall progress, identify at-risk students, and generate detailed reports.

Important Note: This application runs entirely in your web browser. Your data is saved to your browser's local storage. To prevent data loss, regularly use the Export Data function in the Settings tab to create a backup file.

1. Getting Started

🚀

Launching the Application

To begin using the Progress Tracker, simply open the index.html file in any modern web browser (Chrome, Firefox, Edge, or Safari). The application will load immediately with sample data to help you explore its features.

🧭

Navigation

Use the tabs at the top of the page to navigate between the different modules of the application:

  • Dashboard - Overview of key metrics and recent activity
  • Students - Manage student records and progress
  • Courses - Set up and manage course structures
  • Reports - Generate detailed progress reports
  • Settings - Configure application preferences and manage data
Tip: The application comes pre-loaded with sample data for two students and two courses. You can use this to explore features before entering your own data.

2. Dashboard

The Dashboard provides a high-level overview of your institution's activity and performance.

Overview Cards

The default view shows four key metrics cards:

👥

Total Students

The current number of students in the system.

📚

Active Courses

The number of courses that have been created.

✅

Completion Rate

The percentage of students who have completed all tasks in all their enrolled courses.

⚠️

At Risk Students

The number of students who are significantly behind the expected progress timeline in any of their courses.

Charts View

Click the "Charts View" button to toggle from cards to visual data representations:

1 Click the "Charts View" button in the top right of the Dashboard card.
2 Explore the four different charts that appear:
  • Student Distribution: A doughnut chart showing how students are distributed across different courses.
  • Course Progress: A bar chart showing the average percentage of tasks completed for each course.
  • Status Overview: A pie chart showing the proportion of students who are On Track, Behind, At Risk, or have Completed all courses.
  • Monthly Activity: A line graph showing the number of logged activities (task updates) over the past six months.
3 Click the "Cards View" button to return to the metrics cards.

Recent Activity

This table logs the five most recent actions performed in the system, including:

  • Task completions or status changes
  • Student additions or edits
  • New enrollments

For each activity, it shows the date, student name, course, activity description, and status.

3. Students Module

This module is for managing all student records and their enrollments.

Viewing & Filtering Students

The main table lists all students. You can filter this list using the controls above the table:

1 Use the Search Box to type a student's name and find specific students.
2 Use the Course Filter dropdown to select a specific course and show only students enrolled in it.
3 Use the Status Filter dropdown to filter students by their overall status: On Track, Behind, At Risk, or Completed.

Adding a New Student

1 Click the "Add New Student" button.
2 Fill in the required fields: First Name and Last Name.
3 Add optional contact information: Email and Phone.
4 In the Enrolled Courses multi-select box, choose all courses this student is taking.
5 For Rolling Intake Courses: A date field will appear for each rolling intake course you select. Set the student's individual start date for that course.
6 Selecting Units: For each selected course, a list of units will appear. By default, all units are selected. Uncheck any units the student is not required to take.
7 Click Save. The student will be added, and progress tracking will be initialized for all selected units and tasks.

Editing a Student

1 In the Students table, click the "Edit" button next to the student's name.
2 The same modal used for adding a student will open, pre-populated with the student's existing data.
3 Modify any details, enrollments, or unit selections as needed.
4 Click Save. This will update the student's record and adjust their progress data if units were added or removed.

Viewing Student Details & Updating Progress

This is the primary way to update a student's progress.

1 In the Students table, click the "View" button next to the student's name.
2 A detailed modal will open showing the student's contact info, overall status, and a list of their enrolled courses.
3 For each course, you can see the overall progress bar and status.
4 Click on a course unit to expand it and see the individual assessment tasks.
5 To update a task's status, click on the colored circle next to the task name. It will cycle through four statuses:
  • Grey (not-started): Task has not been attempted.
  • Green (completed): Task has been successfully completed.
  • Yellow (partial): Task is partially complete or needs revision.
  • Red (failed): Task was attempted but not passed.
6 The student's progress percentage and status badges will update in real-time as you click. This action is automatically saved and logged in the Recent Activity feed.

Deleting a Student

1 Click the "Delete" button next to the student's name in the table.
2 A confirmation dialog will appear warning you that all of the student's progress data will be permanently removed.
3 Click OK to confirm deletion or Cancel to keep the student.

4. Courses Module

This module is for setting up and managing the structure of your vocational courses.

Viewing Courses

The main table lists all created courses, showing their:

  • Course Code
  • Course Name
  • Duration (weeks)
  • Start Date (or "Rolling Intake")
  • Number of enrolled students
  • Action buttons (Edit, Delete, Units)

Adding a New Course

1 Click the "Add New Course" button.
2 Fill in the required fields:
  • Course Code: A short identifier (e.g., CERT3-IT)
  • Course Name: The full name of the course
  • Duration: The length of the course in weeks
3 Configure the intake:
  • Start Date: Set a fixed start date for the entire course
  • Rolling Intake: Check this box if students can start the course at any time. This will disable the fixed start date and allow you to set individual start dates for each student upon enrollment.
4 Add an optional Description.
5 Click Save. The course will now appear in the table and be available for student enrollment.

Editing a Course

1 In the Courses table, click the "Edit" button for the course you want to modify.
2 Change any of the course details (code, name, duration, intake settings, description).
3 Click Save.

Managing Course Units

Courses are made up of Units, which contain Assessment Tasks.

1 In the Courses table, click the "Units" button for the desired course.
2 The lower section of the page will refresh to show a table of all units within that course.
3 To Add a Unit:
  • Click the "Add Unit" button.
  • Fill in the Unit Code, Unit Name, and Order in Course (a number determining its sequence).
  • Add an optional Description.
  • Add Assessment Tasks: Click the "Add Task" button to create a new field for each assessment task (e.e., "Written Report", "Practical Demo"). Name each task clearly.
  • Click Save. The unit is added to the course.
4 To Edit a Unit: Click the "Edit" button next to the unit in the list. Modify its details or tasks and click Save.
5 To Delete a Unit: Click the "Delete" button next to the unit. Warning: This will also delete all student progress data associated with this unit.

Deleting a Course

1 Click the "Delete" button for the course in the table.
2 A confirmation dialog will warn you that this will also delete all units, tasks, and student progress data for this course.
3 Click OK to confirm or Cancel to abort.

5. Reports Module

This module allows you to generate various reports based on your data.

Generating Reports

1 Select Report Type:
  • Completion Report: Shows completion rates (%) for each course.
  • Progress Report: Lists each student's progress (tasks completed) and status for selected courses.
  • Attendance Report: Shows a log of all student activities within a date range.
  • Task Completion Report: Provides a detailed breakdown, for each task in each unit, of how many students have completed, not started, partially completed, or failed it.
  • Custom Report: Shows a summary of available data.
2 Select Course: Choose a specific course or "All Courses".
3 Set Date Range: For relevant reports (like Attendance), set a start and end date to filter the results.
4 Click the "Generate Report" button.
5 The results will appear in the Report Results section below the button.

6. Settings Module

Application Settings

1 Institution Name: Change the name displayed at the top of the application.
2 Color Theme: Choose from predefined color schemes (Blue, Green, Red, Purple) or select Custom to define your own Primary, Secondary, and Accent colors using the color pickers that appear.
3 Click Save Settings to apply your changes.

Data Management

1 Export Data: Click this button to download a complete backup of all your application data (students, courses, progress, settings) as a .json file. This is your primary backup method.
2 Import Data: Click this button to upload a previously exported .json file and restore your data. Warning: This will completely overwrite all current data in the application. A confirmation dialog will appear.

7. Troubleshooting

❓

"My data is gone!"

The app stores data in your specific browser. If you switch browsers or devices, the data will not be there. Always use Export Data to create a backup you can import elsewhere.

❓

The page looks broken

Ensure you are using a modern, up-to-date browser like Chrome, Firefox, Edge, or Safari.

❓

A feature isn't working

Open your browser's Developer Tools (F12) and check the "Console" tab for any error messages (red text). This can help identify the issue.

❓

I get an alert about an error

The application has built-in error handling. Read the alert message carefully, as it often explains what went wrong (e.g., a required field was left empty).
